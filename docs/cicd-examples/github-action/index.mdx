# Getting started

Say goodbye to bad code and messy multi-repo compliance.
Using our CLI, you easily enforce best practices to organization-wide projects using global code compliances.

Exlint can be used on the command line to run, use and analyze code using customized policy groups.

:::tip

We don't need access to your code or anything whatsoever for you to setup exlint in your projects!
:::

### 2. Install the Exlint CLI

Binary releases can be downloaded from the [releases page](https://github.com/Exlint/CLI/releases) or can be installed directly

<Tabs>
  <TabItem value='Linux' label='Linux' default>

    curl https://get.datree.io | /bin/bash

  </TabItem>
  <TabItem value='MacOS' label='MacOS'>

    curl https://get.datree.io | /bin/bash

  </TabItem>
  <TabItem value='Windows (PowerShell)' label='Windows (PowerShell)'>

    iwr -useb https://get.datree.io/windows_install.ps1 | iex

  </TabItem>
  <TabItem value='Homebrew' label='Homebrew'>

    brew tap datreeio/datree
    brew install datreeio/datree/datree

  </TabItem>
  <TabItem value='Docker' label='Docker'>

    docker pull datree/datree

  </TabItem>
</Tabs>
